# ⚽ ЧМ-2026 — Сайт прогнозов

Платформа прогнозов на матчи Чемпионата мира 2026: общие матчи и результаты,
поверх которых параллельно идут **комнаты** — независимые соревнования со своими
участниками, правилами и таблицами. Бэкенд — FastAPI (async), фронтенд — React
SPA, плюс VK-бот сообщества. Подробная спецификация — в [AGENTS.md](AGENTS.md).

## Возможности

- **Публичное лобби:** список комнат виден без входа; авторизация запрашивается
  только при действии (вступление, прогноз)
- Вход через **Telegram Login Widget** и **Яндекс OAuth 2.0**; первый вошедший
  автоматически становится **суперадмином** (атомарно)
- **Комнаты (несколько соревнований параллельно):** суперадмин создаёт комнаты,
  у каждой свой пароль; участник ищет комнаты и вступает в любое их количество.
  Матчи и результаты общие, а прогнозы, спецпрогнозы и таблица лидеров — свои
  в каждой комнате.
- **Свои правила начисления в каждой комнате:** очки за точный счёт / разницу /
  исход, отдельно за чемпиона и бомбардира; редактируемый текст регламента
  (кнопка «i» у заголовка)
- **Коэффициенты матчей — свои в каждой комнате** (задаёт админ комнаты): ×2 / ×3
  на матч или на весь тур; ×0 — аннулирование (изменение коэффициента у
  завершённого матча пересчитывает очки в этой комнате)
- **Архив комнат:** комната становится доступной только для просмотра
  (начисление прекращается, таблица сохраняется); восстановима
- Прогнозы на счёт матчей (только основное время, 90 мин), дедлайн до `kickoff_at`;
  на странице прогноза — форма обеих сборных (последние матчи 2026 года во всех
  турнирах, пополняется матчами ЧМ по ходу турнира)
- Спецпрогнозы: чемпион турнира + лучший бомбардир (в каждой комнате свои);
  итогового бомбардира назначает админ комнаты, чемпион — автоматически по
  финалу при любом исходе (включая серию пенальти)
- Результат матча вводит только суперадмин, один раз — пересчитывает очки во
  всех комнатах (каждая по своему коэффициенту); исправление счёта корректно
  снимает и начисляет очки заново
- Таблица лидеров с тайбрейком и Redis-кэшем (по комнате); автообновление
  данных на фронте каждые 5 минут
- Вкладка «ЧМ-2026»: реальное положение турнира — таблицы групп (шахматка
  очных встреч, разница мячей, очки), матчи плей-офф по стадиям и блок
  бомбардиров турнира (топ-5 + выбор участников комнаты); над вкладками —
  лента последних начавшихся матчей
- На странице матча после старта — полоса распределения прогнозов (П1 / ничья / П2)
- Профиль игрока в комнате (место, статистика попаданий, история прогнозов);
  свой профиль: никнейм, загрузка аватара, привязка VK
- **VK-бот:** таблица, итоги тура и прогнозы прямо в личке сообщества;
  привязка аккаунта одноразовым кодом из профиля — [BOT_VK.md](BOT_VK.md)
- Polling API-Football (APScheduler, каждые 5 минут в игровые дни) + ручной
  ввод результатов
- Глобальная панель **(только суперадмин):** матчи / результаты / синхронизация /
  пересчёт / журнал с CSV-экспортом. Управление комнатой (RAdmin): участники /
  пароль / регламент / коэффициенты матчей / итоговый бомбардир
- **Режим симуляции (суперадмин):** посмотреть, как будут выглядеть экраны
  в будущий момент («матч начался», «3 тура сыграны», «турнир завершён») —
  фейковые результаты и очки считаются на лету, строго без записи в БД
- **Суперадмин по умолчанию видит сайт как обычный игрок** — без чужих
  прогнозов до дедлайна и без админ-кнопок в комнатах; полные права
  включаются чекбоксом «Режим суперадмина» в глобальной панели
  (с баннером-уведомлением)

## Стек

| Слой | Технологии |
|------|-----------|
| Бэкенд | FastAPI, SQLAlchemy 2 async, Alembic, PostgreSQL 15, Redis 7, APScheduler, httpx |
| Фронтенд | React 18, TypeScript, Vite, Tailwind, TanStack Query, Zustand, React Router |
| Инфра | Docker, Docker Compose |

## Структура

```
backend/    FastAPI приложение, миграции, тесты, seed, ядро ботов
frontend/   React SPA
docker-compose.yml   db + redis + backend + frontend
AGENTS.md   полная спецификация (источник правды)
DEPLOY.md   инструкция по размещению на сервере
BOT_VK.md   настройка VK-бота
```

## Быстрый старт (локально через Docker)

```bash
cp backend/.env.example backend/.env     # заполнить секреты
docker compose up --build -d
docker compose exec backend python -m scripts.seed   # матчи + первая комната с паролем
```

Фронтенд: http://localhost:5173 · API-документация: http://localhost:8000/docs

## Тесты

```bash
cd backend
pip install -r requirements.txt
pytest                # scoring.py и Telegram HMAC — без БД
```

Полная инструкция по развёртыванию на сервере — **[DEPLOY.md](DEPLOY.md)**.
