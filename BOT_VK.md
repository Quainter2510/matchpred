# 🤖 VK-бот · прогнозы

Бот сообщества ВКонтакте для прогнозов: посмотреть таблицу, итоги тура и сделать
прогноз — прямо в личке сообщества. Авторизация без VK ID: бот узнаёт VK-id из
события `message_new`, а аккаунт привязывается одноразовым кодом из профиля.

Бот работает с **любыми типами турниров** (ЧМ/РПЛ/ЛЧ/custom): все чтения матчей
скоупятся по выбранному турниру (`queries.tournament_match_conditions`), а меню
показывает короткую метку типа (ЧМ/РПЛ/ЛЧ/Свой).

## Возможности
- Привязка аккаунта по коду (код выдаёт сайт в профиле).
- Если у игрока один активный турнир — сразу меню; если несколько — выбор кнопками (с меткой типа).
- **Таблица** — участники и очки, сортировка по убыванию.
- **Итог тура** — выбор тура; таблица очков за тур + матчи с вашими очками.
- **Сделать прогноз** — выбор тура; матчи по очереди, счёт вводится как «2 1».

Листинг туров учитывает схему турнира: у суточных (ЧМ) окно узкое (−3…+2 дня),
у недельных (РПЛ/ЛЧ/custom) — шире (тур = неделя). Окно применяется к `kickoff`
матчей, поэтому текущий недельный тур не теряется, даже если его дата-метка
(начало недели) уже прошла.

## Настройка сообщества VK

1. Создайте сообщество (паблик/группа).
2. **Управление → Сообщения** → включить сообщения сообщества.
3. **Управление → Работа с API → Ключи доступа** → создать ключ сообщества с
   правом «Управление сообществом» и «Сообщения» → это `VK_GROUP_TOKEN`.
4. **Работа с API → Callback API**:
   - Тип события: включить **«Входящее сообщение»** (`message_new`).
   - Версия API: `5.199` (или актуальная) → `VK_API_VERSION`.
   - Адрес сервера: `https://ВАШ_ДОМЕН/api/v1/bots/vk/callback`.
   - **Секретный ключ** → задайте и продублируйте в `VK_SECRET`.
   - Нажмите «Подтвердить». VK покажет **строку подтверждения** → впишите её в
     `VK_CONFIRMATION` (бэкенд должен быть уже задеплоен с этой переменной, чтобы
     ответить на проверку).
5. `VK_GROUP_ID` — числовой id сообщества (без минуса).

## Переменные окружения (`backend/.env`)
```env
VK_GROUP_TOKEN=ключ-доступа-сообщества
VK_GROUP_ID=123456789
VK_CONFIRMATION=строка-подтверждения-из-vk
VK_SECRET=секретный-ключ-callback
VK_API_VERSION=5.199
```

После изменения `.env`:
```bash
docker compose up -d backend
```

## Порядок подключения
1. Задеплойте бэкенд с `VK_CONFIRMATION` (и остальными VK-переменными).
2. В Callback API нажмите «Подтвердить» — должно стать «OK».
3. Включите событие `message_new`.
4. Напишите сообществу из тестового аккаунта — бот попросит код привязки.

## Привязка аккаунта
Кнопка привязки — на странице **Профиль** на сайте:
- Кнопка вызывает `POST /api/v1/auth/vk/link-code` (под JWT) → `{ "code": "A3F9C1", "bot_url": "https://vk.me/clubXX…" }`.
- Игрок отправляет код боту → бот привязывает VK к аккаунту (`oauth_accounts`, provider=`vk`).
- Статус привязки виден в профиле (`vk_linked` в `GET /auth/me`); привязку можно выполнить заново.

Код одноразовый, живёт 10 минут (Redis).

## Архитектура (для будущего Telegram-бота)
Логика бота транспортно-независима:
- `app/services/bot/core.py` — меню, выбор комнаты, три сценария, конечный автомат
  (состояние диалога в Redis).
- `app/services/bot/queries.py` — read-запросы.
- `app/services/bot/vk_client.py` — отправка сообщений в VK.
- `app/routers/bots.py` — VK Callback (адаптер: событие ↔ ядро, рендер клавиатуры).

Сохранение прогноза — общий сервис `app/services/predictions.py::set_prediction`
(те же правила дедлайна/архива/журнала, что и в REST). Для Telegram добавится
вебхук + адаптер, ядро не меняется.
